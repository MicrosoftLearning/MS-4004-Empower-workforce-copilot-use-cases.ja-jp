# 演習 2、タスク 1: Microsoft 365 Copilot Chat を使って Microsoft 365 の今後の機能を明らかにする
---
VanArsdel の次の IT リーダーシップ会議の準備として、あなたは次の四半期にユーザーに影響する可能性がある今後の Microsoft 365 の機能の概要を作成する必要があります。 IT チームがコミュニケーションとサポートの戦略を計画できるように、この情報はすぐに必要です。 目標は、今後 90 日以内に VanArsdel のエンド ユーザーに影響を与える可能性のある最も関連性の高い新機能を明らかにしてまとめ、IT 部門の対応の可能性に関する注意を示すことです。

これまでのこの作業では、チームは、ブログと管理センターの更新を検討し、結果を統合し、エンド ユーザーに対する各機能の影響を評価して、IT 部門での運用の効果を評価してから、プレゼンテーションのために最も重要な 8 つの項目の優先順位を付ける必要がありました。 ここでは、数時間かけてこの作業を終わらせる代わりに、Microsoft 365 Copilot Chat を使って今後の見通しを収集して要約する予定です。すべてを約 1 分で行い、Copilot を使うと時間が劇的に節約をされることを見ていきます。

#### Copilot Chat を使う

Web 上の Copilot Chat では、応答モード セレクターを使用して、プロンプトに応答するときに Copilot が使用する時間と推論の量を制御できます。 Copilot が速度と深度のバランスを取るように **[Auto]** (既定のオプション) に設定したままにするか、タスクに応じてより速く、またはより深い応答スタイルを選択することができます。

Copilot Chat が **[作業]** モードで開かれると、応答モード セレクターは表示されません。 **[作業]** モードでは、Copilot はセキュリティで保護された作業コンテキスト クエリ用に最適化されているため、応答の深さを自動的に管理します。 **[Web]** モードに切り替えると、応答モード セレクターが表示され、より高速な応答またはより深い推論を選択できます。 セレクターが有効になった後、**[作業]** モードと **[Web]** モードを切り替えるとセレクターが表示されたままになります。

[Copilot in Excel] を使用したことがある場合は、応答コントロール セレクターも含まれていることがわかります。 ただし、そのオプションはチャット セレクターとは異なります。 Copilot Chat では、セレクターによって、要求に関して Copilot がどれだけ深く推論するかが制御されます。 Excel では、セレクターによって、作業を実行する AI モデルが制御されます。 これらのセレクターは似ているかもしれませんが、Copilot の異なる側面を制御するものであり、同じ設定ではありません。

このタスクでは、既定の **[Auto]** セレクター モードが使用されます。

このタスクを完了するには、次の手順を実行します。

1.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインします。 このシナリオでは、Microsoft によって公開され、VanArsdel のテナントに格納されていない将来の機能を検索する必要があるため、**[作業]** オプションではなく **[Web]** オプションを選びます。 応答モード セレクターは **[Auto]** に設定したままにします。

2.  今後の Microsoft 365 の機能を要約するよう Copilot に指示する次のプロンプトをコピーして貼り付けます。
  
   > [!NOTE]
   > この最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素が組み込まれている場合に、効果的なプロンプトがどのようになるかわかるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて学習者が記述する必要がありますが、その際は、このプロンプトを手本として参考にできます。

   **私は VanArsdel, Ltd. の IT マネージャーです。今後 90 日以内に Microsoft によってリリースまたは一般提供される可能性が高い Microsoft 365 の機能のうち、エンド ユーザーに最も関連性の高いもの (コラボレーション、セキュリティ、Teams、Outlook、SharePoint、Viva など) をまとめています。各機能と共に、1 行の説明、技術部門以外のエンド ユーザーにとって重要な理由、IT 部門に推奨される対応 (コミュニケーション、トレーニング、テナント設定の変更など) を一覧にしてください。結果は短い箇条書きとして提供します (8 項目以下)。**

3.  Copilot の応答を検討します。 ソースが返された場合は、そのうちの 2 つを調べて、それらが有効であることを確認します。 あいまいな項目が返された場合は、チャットで明確化のためのフォローアップ質問をします。

4.  応答の最後に、**[Pages で編集]** アイコンを選択します。

5.  **Pages** での編集中に、Copilot によって Pages フォームと共に [Copilot Chat] ペインが表示されることに注目してください。 **Pages** フォームで **[作成]** ボタンを選択し、ドロップダウン メニューから **[文書]** を選択します。 表示されるダイアログ ボックスで、**[Word を開く]** を選択すると、文書が **Word for the web** で開きます。 

6.  **Word for the web** では、チャットの最初と最後に表示された不要なチャット内容を含め、応答全体が Copilot によって文書にコピーされます。 貼り付けられたテキストに不要なものがあれば削除します。 コミュニケーションで優先する項目をタグで示すか強調表示にします。 

7.  Word 文書で、**[ホーム]** タブの **[Copilot]** オプションを選択します。プロンプト フィールドにプラス (+) 記号の横に **[Copilot で編集]** アイコンが表示されていることを確認します。 表示されていない場合は、プラス記号を選択し、ドロップダウン メニューから **[Copilot で編集]** を選択します。 プロンプト フィールドにアイコンが表示されます。

8.  次の手順では、文書に概要を追加するよう Copilot に依頼します。 その前に、まず Copilot による要約の挿入場所にカーソルを置く必要があります。 要約を機能一覧の前に表示する場合は、文書のタイトルと機能一覧の間にカーソルを移動します。 

9.  [Copilot Chat] ペインで、文書の内容に基づいて短い要約を追加するよう Copilot に依頼します。 
 
10. Copilot によって要約が文書に追加されたら、各機能について 1 文のリスクまたは機会を追加するよう依頼します。 

11. コンテンツを確認します。 リスクと機会が、各機能ではなく別のセクションに Copilot によって追加された場合は、追加されたリスクと機会のセクションを削除し、リスクまたは機会を別のセクションではなく各機能の末尾に追加するように依頼します。 
 
12. 新しい内容を確認します。 Copilot によって作成された新しい内容の書式に満足できない場合は、必要に応じて調整を加えます。

13. タスク 3 でこの文書を使用する予定なので、メニュー バーの上に表示されるファイル名をメモしておきます。
