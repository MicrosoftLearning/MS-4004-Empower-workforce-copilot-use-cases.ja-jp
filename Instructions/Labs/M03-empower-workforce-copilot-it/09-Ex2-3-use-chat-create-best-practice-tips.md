# 演習 2、タスク 3:Microsoft 365 Copilot Chat を使用してベスト プラクティスのヒントを作成する
---
お知らせの投稿の準備ができたので、ユーザーがこれらの機能をスムーズに採用できるように支援したいと考えています。 そのために、VanArsdel のエンド ユーザーがトレーニング セッション中に共有できるクイック ガイドを作成します。 また、このクイック ガイドをすべてのオンボーディング資料に含めます。

ゼロから作り始めるのではなく、先ほど調査したのと同じ機能に基づいて、これらの実用的で読みやすいヒントを生成するように Copilot Chat に依頼する予定です。 あなたの目標は、従業員が自信を持って新しい Microsoft 365 機能を使用し始めるのに役立つ、10 個の短く実践的なベスト プラクティスのヒントを作成することです。

このタスクを完了するには、次の手順を実行します。

1.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページにサインインします。 このシナリオでは、Microsoft によって公開され、VanArsdel のテナントには格納されていない将来の機能を実装する必要があるため、**[Work]** オプションではなく **[Web]** オプションを選びます。 応答モード セレクターは **[Auto]** に設定したままにします。

2.  Copilot Chat プロンプトで、Microsoft 365 機能リストを含むタスク 1 のドキュメントを添付します。 その後、Copilot に「**10 のベスト プラクティスのヒント**」というクイック ガイドを作成するように依頼します。 添付されているファイルに記載されている新しい Microsoft 365 機能を採用するための 10 個の簡潔なベスト プラクティスのヒントが含まれている必要があります。 各ヒントには、ヒントの内容を説明する短い文と、その後に、ヒントがユーザーにとって重要な理由を説明する短い説明 (最大 25 語) が含まれている必要があります。 

3.  Copilot で生成されたクイック ガイドを確認します。 興味のある Copilot プロンプトの候補を送信してみてください。 たとえば、ポスター風のビジュアルを追加するよう Copilot に依頼したい場合があります。 

4.  ヒントを以下の形式で出力するよう Copilot に依頼します。
    - 一覧を 1 ページのチート シート (Word 文書) と PDF に変換してください。
    - 一覧をメール用のバージョンに変換してください (短い件名の行と 5 つの箇条書きを含めてください)。

5.  Copilot によって生成された Word 文書と PDF をダウンロードします。 各ファイルを開いて内容を確認します。

6.  この資料を現場担当者の教育に使用する予定でしたが、経営幹部、営業担当者、IT 担当者向けに調整した別バージョンを作成する方が良いかもしれないと気付きます。 経営幹部、営業担当者、IT 担当者向けに特化したペルソナ別のチート シートを PDF 形式で作成するよう Copilot に依頼します。

7.  各 PDF ファイルをダウンロードして内容を確認します。 各ペルソナの文書の違いに注目してください。
