# 演習 1、タスク 1: Copilot Chat を使って、プロジェクトのフレームワークを定義する
---
このタスクでは、Copilot Chat によってどのようにプロジェクト計画の初期段階が速まり、IT、ビジネス、リーダーシップのチーム間の連携が実現されるかを見ていきます。

あなたは、Boulder Innovation の IT チームのリーダー メンバーであり、ネットワーク最新化とセキュリティ アップグレード プロジェクトの実装を担当しています。 作業を始める前に、CIO はプロジェクトの目的、目標、成果物、利害関係者をよく理解したいと考えています。 CIO はあなたのチームに、プロジェクトのドキュメント、メール スレッド、チャット ログからすべての関連情報を収集してまとめ、チームの計画と実行の指針となる一貫したプロジェクト フレームワークを作成するよう指示しました。

このタスクを完了するため、あなたは Microsoft 365 Copilot Chat を使って、組織全体の複数のソースから情報を収集してまとめる予定です。 このシナリオでの Copilot Chat は、ばらばらのデータ (メール、Teams ディスカッション、プロジェクト ドキュメント) 間の橋渡しとして機能し、IT チームがプロジェクトの要件と利害関係者の期待の統一されたビューをすばやく生成できるようにします。

> [!NOTE]
> このコースはサブスクリプション持ち込み (BYOS) モデルに基づいているため、Boulder Innovations 用のシミュレートされたラボやデモ環境は使いません。 そのため、アクセスする Microsoft 365 システムはありません。 このような状況に対処するために、このラボでは、Word (.docx) ファイルで提供されるシミュレートされたメールを使用します。 それには、送信者、受信者、件名、日付、メッセージの内容が含まれます。 このタスクでは Word 文書を使ってメール スレッドをエミュレートしますが、Copilot Chat は実際のメールと同じようにその内容を読んで要約できます。 同様に、簡単でアクセス可能な形式で実際のプロジェクト ディスカッションをシミュレートするため、Teams チャット メッセージがテキスト (.txt) ファイルで提供されます。 実際の企業環境では、これらの会話は Teams に保存されますが、このトレーニング演習ではテキスト ファイルを使うので、すべての学習者はライブ Teams チャネルにアクセスしなくても、Copilot Chat で内容をアップロードして使用できます。 この形式では、送信者、タイムスタンプ、メッセージの内容が保持されるため、実際のチャット メッセージと同様に、重要な情報の正確な要約と抽出を練習できます。  

#### Copilot Chat を使う

Web 上の Copilot Chat では、応答モード セレクターを使用して、プロンプトに応答するときに Copilot が使用する時間と推論の量を制御できます。 Copilot が速度と深度のバランスを取るようにこれを **[Auto]** (既定のオプション) に設定したままにすることも、タスクに応じてより速く、またはより深い応答スタイルを選択することもできます。

Copilot Chat が **[作業]** モードで開かれると、応答モード セレクターは表示されません。 **[作業]** モードでは、Copilot はセキュリティで保護された作業コンテキスト クエリ用に最適化されているため、応答の深さを自動的に管理します。 **[Web]** モードに切り替えると、応答モード セレクターが表示され、より高速な応答またはより深い推論を選択できます。 セレクターが有効になった後、**[作業]** モードと **[Web]** モードを切り替えるとセレクターが表示されたままになります。

[Copilot in Excel] を使用したことがある場合は、応答コントロール セレクターも含まれていることがわかります。 ただし、そのオプションはチャット セレクターとは異なります。 Copilot Chat では、セレクターによって、要求に関して Copilot がどれだけ深く推論するかが制御されます。 Excel では、セレクターによって、作業を実行する AI モデルが制御されます。 これらのセレクターは似ているかもしれませんが、Copilot の異なる側面を制御するものであり、同じ設定ではありません。

このタスクでは **[作業]** モードが使用されるため、応答モード セレクターは適用されません。

このタスクを完了するには、次の手順を実行します。

1.  次のドキュメントでは、ネットワーク最新化とセキュリティ アップグレード プロジェクトを参照している、Boulder Innovations (BI) 全体のさまざまなメール、Teams メッセージ、ドキュメントがエミュレートされています。 次の各リンクを選んでそれぞれのファイルをダウンロードし、OneDrive アカウントに保存します。
    - [**BI Project Proposal Draft.docx**](https://go.microsoft.com/fwlink/?linkid=2347512)
    - [**BI Email Thread – Budget Discussion.docx**](https://go.microsoft.com/fwlink/?linkid=2347806)
    - [**BI Teams Chat Export.txt**](https://go.microsoft.com/fwlink/?linkid=2347513)
    - [**BI Meeting Notes – Stakeholder Brief.docx**](https://go.microsoft.com/fwlink/?linkid=2347511)
    - [**BI Email – Vendor Quote Summary.docx**](https://go.microsoft.com/fwlink/?linkid=2347510)
    - [**BI Security Risk Memo.docx**](https://go.microsoft.com/fwlink/?linkid=2347807)
        
2.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインします。

3.  Microsoft 365 Copilot Chat で、**[作業]** タブが選択されていることを確認します。 プロンプト フィールドで、手順 1 でダウンロードした各ファイルを添付します。 これらのソースからすべてのプロジェクト要件を要約するよう、Copilot に指示します。 主要な利害関係者を特定し、プロジェクトの目標、目的、期待される成果物の概要を示す必要があります。 経営陣と共有できる簡潔なプロジェクト フレームワークの概要を作成させます。 また、各セクションをソースにマップする必要もあります。
  
   > [!NOTE]
   > この最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素が組み込まれている場合に、効果的なプロンプトがどのようになるかわかるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて学習者が記述する必要がありますが、その際は、このプロンプトを手本として参考にできます。

   **添付された文書を使って、Boulder Innovation のネットワーク最新化とセキュリティ アップグレード プロジェクトをまとめます。目的と範囲、主要な目標、重要な成果物、マイルストーン、リスク、利害関係者など、経営陣向けの簡潔なフレームワークを作成します。ソースを引用し (ファイル名と日付)、競合する詳細を注記します。短いセクションと箇条書きで概要を示し、その後に各セクションをソースにマップする付録を付けます。**

4.  結果を確認します。 終わったら、ダウンロードして CIO と共有できる書式設定された Word 文書に結果を変換するよう、Copilot に指示します。

5.  Copilot によって生成された文書をダウンロードし、**ダウンロード** フォルダーから **OneDrive** フォルダーに移動します。

6.  Word でファイルを開き、確認します。 要求したすべての情報が含まれていることを確認します。

7.  **[ホーム]** タブ リボンで **[Copilot]** を選択します。 Copilot ペインで、プロンプト フィールドのプラス (+) 記号の横に **[Copilot で編集]** アイコンが表示されていることを確認します。 表示されていない場合は、プラス記号を選択し、ドロップダウン メニューで **[Copilot で編集]** を選択します。 プロンプト フィールドにアイコンが表示されます。

8.  文書を検討した後、視覚エフェクトをいくつか使うと一部のセクションをもっとよくできると思われます。 文書の内容を補強できる画像、グラフ、または SmartArt グラフィックを提案するよう、Copilot に指示します。

9.  提案の一覧から、Copilot に生成させたいものを選んで、その画像の生成を指示します。

   > [!NOTE]
   > 時間の制約があり、Copilot が画像を生成するには時間がかかるので、要求は 1 つの画像に限らせてもらいます。

10. エージェント モードが有効になっているときに (または、バージョンによっては **[Copilot で編集]** と呼ばれている場合があります) イメージを生成または挿入できないことが Copilot で示されている場合は、プロンプト フィールドの **[Copilot で編集]** アイコンを選択します。 これにより、**[Copilot で編集]** モードが終了します。 次に、選択した画像を生成するよう Copilot に指示するプロンプトを再送信します。

11. Copilot で画像を挿入する文書にカーソルを置きます。 Copilot が Copilot ペインで画像を生成したら、画像の上にマウス ポインターを合わせ、表示されるプラス (+) 記号を選択します。 この操作を行うと、画像が文書のカーソルの位置に挿入されます。 

12. **[Copilot で編集]** モードを終了して画像を生成する必要がある場合は、プロンプト フィールドでプラス記号を選択し、ドロップダウン メニューで **[Copilot で編集]** を選択して、**[Copilot で編集]** モードに戻ります。

13.  最後に、質問と回答 (Q&A) のセクションは経営陣に役立つと思われます。 文書の末尾にカーソルを置きます。この位置に、Copilot で Q&A セクションを挿入します。 次に、文書の内容に基づいて潜在的な質問と回答を含む Q&A セクションを文書に追加するよう Copilot に指示します。

14.  Copilot が文書に挿入した Q&A セクションを確認します。 問題ないようですが、一部の質問と回答は、単に既存の文書の箇条書きを言い換えているだけだと感じます。 Q&A 資料を強化するには、より深い思考を誘発する、前提条件を疑う、またはプロジェクトをより広範なコンテキストに結び付けるさまざまな一連の質問と回答を生成するよう Copilot に指示します。

15.  強化された Q&A セクションを確認します。 質問とインサイトが深いほど、好みに合うはずです。 この文書をまとめる前に、Copilot ペインの最後に表示されるプロンプトの候補を確認します。 提案された変更のいずれかを行う場合は、関連付けられているプロンプトを選択し、今すぐ要求を送信します。

16.  この文書は、タスク 3 でエグゼクティブ プレゼンテーションの基礎として使用する予定なので、OneDrive 内のファイル名をメモしておいてください。
