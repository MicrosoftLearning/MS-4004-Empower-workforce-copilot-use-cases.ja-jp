# 演習 1、タスク 1: Copilot Chat を使って、プロジェクトのフレームワークを定義する
---
このタスクでは、Copilot Chat によってどのようにプロジェクト計画の初期段階が速まり、IT、ビジネス、リーダーシップのチーム間の連携が実現されるかを見ていきます。

あなたは、Boulder Innovation の IT チームのリーダー メンバーであり、ネットワーク最新化とセキュリティ アップグレード プロジェクトの実装を担当しています。 作業を始める前に、CIO はプロジェクトの目的、目標、成果物、利害関係者をよく理解したいと考えています。 CIO はあなたのチームに、プロジェクトのドキュメント、メール スレッド、チャット ログからすべての関連情報を収集してまとめ、チームの計画と実行の指針となる一貫したプロジェクト フレームワークを作成するよう指示しました。

このタスクを完了するため、あなたは Microsoft 365 Copilot Chat を使って、組織全体の複数のソースから情報を収集してまとめる予定です。 このシナリオでの Copilot Chat は、ばらばらのデータ (メール、Teams ディスカッション、プロジェクト ドキュメント) 間の橋渡しとして機能し、IT チームがプロジェクトの要件と利害関係者の期待の統一されたビューをすばやく生成できるようにします。

> [!NOTE]
> このコースはサブスクリプション持ち込み (BYOS) モデルに基づいているため、Boulder Innovations 用のシミュレートされたラボやデモ環境は使いません。 そのため、アクセスする Microsoft 365 システムはありません。 このラボでは、Word (.docx) ファイルで提供されるシミュレートされたメールを使って、このような状況に対処します。 それには、送信者、受信者、件名、日付、メッセージの内容が含まれます。 このタスクでは Word 文書を使ってメール スレッドをエミュレートしますが、Copilot Chat は実際のメールと同じようにその内容を読んで要約できます。 同様に、簡単でアクセス可能な形式で実際のプロジェクト ディスカッションをシミュレートするため、Teams チャット メッセージがテキスト (.txt) ファイルで提供されます。 実際の企業環境では、これらの会話は Teams に保存されますが、このトレーニング演習ではテキスト ファイルを使うので、すべての学習者はライブ Teams チャネルにアクセスしなくても、Copilot Chat で内容をアップロードして使用できます。 この形式では、送信者、タイムスタンプ、メッセージの内容が保持されるため、実際のチャット メッセージと同様に、重要な情報の正確な要約と抽出を練習できます。  

このタスクを完了するには、次の手順を実行します。

1.  次のドキュメントでは、ネットワーク最新化とセキュリティ アップグレード プロジェクトを参照している、Boulder Innovations (BI) 全体のさまざまなメール、Teams メッセージ、ドキュメントがエミュレートされています。 次の各リンクを選んでそれぞれのファイルをダウンロードし、OneDrive アカウントに保存します。
    - [**BI Project Proposal Draft.docx**](https://go.microsoft.com/fwlink/?linkid=2347512)
    - [**BI Email Thread – Budget Discussion.docx**](https://go.microsoft.com/fwlink/?linkid=2347806)
    - [**BI Teams Chat Export.txt**](https://go.microsoft.com/fwlink/?linkid=2347513)
    - [**BI Meeting Notes – Stakeholder Brief.docx**](https://go.microsoft.com/fwlink/?linkid=2347511)
    - [**BI Email – Vendor Quote Summary.docx**](https://go.microsoft.com/fwlink/?linkid=2347510)
    - [**BI Security Risk Memo.docx**](https://go.microsoft.com/fwlink/?linkid=2347807)
        
2.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインします。

3.  これらのソースからすべてのプロジェクト要件を要約するよう、Microsoft 365 Copilot Chat に指示します。 主要な利害関係者を特定し、プロジェクトの目標、目的、期待される成果物の概要を示す必要があります。 経営陣と共有できる簡潔なプロジェクト フレームワークの概要を作成させます。 また、各セクションをソースにマップする必要もあります。
  
   > [!NOTE]
   > この最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素が組み込まれている場合に、効果的なプロンプトがどのようになるかわかるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて学習者が記述する必要がありますが、その際は、このプロンプトを手本として参考にできます。

   **添付された文書を使って、Boulder Innovation のネットワーク最新化とセキュリティ アップグレード プロジェクトをまとめます。目的と範囲、主要な目標、重要な成果物、マイルストーン、リスク、利害関係者など、経営陣向けの簡潔なフレームワークを作成します。ソースを引用し (ファイル名と日付)、競合する詳細を注記します。短いセクションと箇条書きで概要を示し、その後に各セクションをソースにマップする付録を付けます。**

4.  結果を確認します。 終わったら、ダウンロードして CIO と共有できる書式設定された Word 文書に結果を変換するよう、Copilot に指示します。

5.  Copilot によって生成された文書をダウンロードし、**ダウンロード** フォルダーから **OneDrive** フォルダーに移動します。

6.  Word でファイルを開き、確認します。 要求したすべての情報が含まれていることを確認します。

7.  文書を検討した結果、視覚エフェクトをいくつか使うと一部のセクションをもっとよくできると思われます。 内容を補強できる画像、グラフ、または SmartArt グラフィックを提案するよう、Copilot in Word に指示します。

8.  提案の一覧から、Copilot に生成させたいものを選んで、その画像の生成を指示します。

   > [!NOTE]
   > 時間の制約があり、Copilot が画像を生成するには時間がかかるので、要求は 1 つの画像に限らせてもらいます。

9.  Copilot が要求した画像を生成したら、画像を文書に挿入するように指示します。 もともと、Copilot in Word には画像を文書に直接挿入する機能はありませんでした。 ただし、Copilot in Word への機能の追加が続けられているので、この演習を実行する時点では、画像を挿入できるようになっているかもしれません。 まだ画像を挿入できない場合、文書に画像を挿入したければ、用意されている手順に従って自由に行ってください。

10.  最後に、質問と回答 (Q&A) のセクションは経営陣に役立つと思われます。 Q&A セクションを作成し、調査またはレビュー用の文書の内容に基づいて可能性のある質問と回答を生成するよう、Copilot に指示します。

11.  Copilot ペインで Copilot が生成した Q&A セクションを確認します。 問題がなければ、次のステップに進みます。 一方、質問と回答が既存の文書の箇条書きを言い換えているだけであると思われる場合は、考えを深めたり、仮定を疑ったり、プロジェクトをより広いコンテキストに結び付けたりする別の質問のセットを生成するよう、Copilot に指示します。

12.  一連の質問と回答に満足したら、**[文書に追加] (+)** アイコンを選びます。

   > [!CAUTION]
   > **[文書に追加] (+)** アイコンを選ぶと、Copilot は文書内のカーソル位置に内容を挿入します。 したがって、**[文書に追加] (+)** アイコンを選ぶ前に、新しい Q&A セクションを挿入したい場所 (ソース マッピングの付録の前または後など) にカーソルがあることを確認します。

13.  タスク 3 ではエグゼクティブ プレゼンテーションの基礎としてこの文書を使う予定です。
