# 演習 2、タスク 1: Copilot in Loop を使用してトラブルシューティングのチェックリストを作成する
---
あなたは Tailwind Traders のカスタマーサービス マネージャーです。最近、会社の顧客ポータルを使用する複数の顧客に製品保証の検証に関する問題が増加していることに気付きました。 これらの問題は、シリアル番号の不一致から購入証明スキャンの欠落まで多岐に及び、それぞれが顧客満足度に影響を与え、サポート チームの負担を増やしています。 状況に迅速かつ一貫した方法で対処するには、他の部署も貢献し、頼ることができる、統一されたトラブルシューティング ガイドの作成を主導する必要があります。

このガイドを共同で作成し、柔軟性を持たせ、常に最新の状態に保つために、Copilot in Microsoft Loop を使用する予定です。 サポート、保証運用、製品エンジニアリングからの入力が必要なとき、Loop は、製品保証の検証手順を説明するライブ ドキュメントを共同作成するための理想的なスペースになります。 Copilot を使用すると、コンテンツを構造化し、チームごとに責任を割り当て、全員がリアル タイムで貢献できるチェックリストを作成できます。 あなたの目標は、社内のチームの連携と効率性を維持しながら、顧客が迅速かつ正確なサポートを受けられるようにすることです。

このタスクを完了するには、次の手順を実行します。

1.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインし、ナビゲーション ウィンドウで **[アプリ]** を選んで、**[アプリ]** メニューから **[Loop]** を選びます。

2.  **Loop for the web** で、「**トラブルシューティング ガイド**」というタイトルの新しいワークスペースを作成します。

3.  これで、新しいワークスペースの最初のページに移動します。 最初に、先頭ページの現在のタイトルを **[無題]** から「**製品保証トラブルシューティング ガイド**」に変更します。 ナビゲーション ウィンドウでページ名が自動的に更新されることに注意してください。

4.  このページの **Copilot** ペインを開きます。

5.  **Copilot** プロンプトで、製品保証の検証に関する問題を解決するための製品保証検証トラブルシューティング チェックリストを作成するよう Copilot に依頼します。

6.  Copilot で生成された項目の一覧を確認します。 項目は手順ごとに分類する必要があります。

    - Copilot が項目を手順ごとに分割した場合は、Copilot の応答の最後に表示される **[コピー]** アイコンを選択し、Loop ページを選択してコンテンツを貼り付けます **(Ctrl + V)**。 会話のコンテンツから、共にコピーして貼り付けた余分なコンテンツを削除します (通常はコンテンツの先頭と末尾)。

    - Copilot がトラブルシューティング プロセスを明確な手順に分割しなかった場合は、それを行うように依頼します。 応答をコピーして Loop ページに貼り付け、コンテンツの先頭と末尾に表示される不要なテキストを削除します。

7.  Copilot ペインで Copilot の応答を確認します。 順調なスタートを切ったようですが、各手順に関連付けられたタスクをもう少し詳しく把握しておきたいところです。 そこで、Copilot に対して、生成されたものは良いスタートだが、一部のタスクは大まかすぎることを伝えます。 各手順で特定されたタスクについて、さらに詳しく記述するように依頼します。

8.  Copilot の回答を確認します。 以前より適切になりましたが、まだ何かが欠けているように感じます。 各手順の責任者がどこにも記載されていません。 この不足に対処するため、各手順を担当するチーム (サポート、保証運用、製品エンジニアリングなど) を特定するよう Copilot に依頼します。

9.  応答をコピーして Loop ページに貼り付け、元のチェックリストの内容を置き換えます。 コンテンツの先頭と末尾に表示されている余分なテキストを削除します。

10. Loop ページで最後にもう一度一覧を確認すると、重要な連絡先情報と会社のエスカレーション ポリシーが記載されていないことに気付きました。 これらの不足している部分に対処するために、主要な連絡先情報と会社のエスカレーション ポリシーを記載したセクションを上部に追加するよう Copilot に依頼します。

11. 以前のプロンプトでは、Copilot は常に更新されたコンテンツと共にリスト全体を返していました。 ただし、このプロンプトでは、Copilot は連絡先情報を含むセクションを表示しただけです。 このコンテンツをコピーし、Loop ページのページ上部に貼り付けます。 これは、ページ上のコンテンツの最初のセクションである必要があります。 コンテンツの先頭と末尾に表示されている余分なテキストを削除します。

12. チームの他のメンバーと相談した後、変更をもう 1 つ加えることにしました。 各チームにとってより有用なドキュメントにするために、Copilot を使用してドキュメントを単純な一覧から Loop チェックリスト コンポーネントに変更したいところです。 こうすることで、ユーザーは各手順が完了するごとにマークできるようになります。 この変換を行うには、チームがリアルタイムで更新できるように、このリストを Loop チェックリスト コンポーネントに変換するよう Copilot に依頼します。

13. 応答をコピーして Loop ページに貼り付け、現在のチェックリストの内容を置き換えます。 コンテンツの先頭と末尾に表示されている余分なテキストを削除します。

14. これで、チェックリストは満足できるものになりました。 ただし、念のために、このチェックリストに追加する必要がある他のものがあるかどうかを Copilot に問い合わせて、より効果的な製品保証トラブルシューティング ガイドにしましょう。

15. Copilot の応答を検討します。 問題なさそうなので、この内容をコピーし、Loop ページのページ末尾に貼り付けます。 これは、ページ上のコンテンツの最後のセクションである必要があります。 コンテンツの先頭と末尾に表示されている余分なテキストを削除します。
