# 演習 2、タスク 4: Copilot Chat および Copilot in Outlook を使用してサポート分析情報メールを作成する
---
最近、Tailwind Traders のリーダーシップ チームは、ディーラー ネットワーク全体における顧客対応の傾向をより明確に把握する必要性を強調しました。 毎週、多数の製品問い合わせ、保証申請、トラブルシューティング依頼、品質に関する懸念がサポート部門に寄せられています。 このようなやり取りには、繰り返し発生する障害点、新しい品質傾向、季節的な急増、トレーニング不足などの貴重なシグナルが含まれています。 ただし、分析情報の多くは個別のチケットや会話の中に埋もれたままです。

リーダーシップ レベルでのより適切な意思決定を支援するために、あなたは、過去 3 か月間のサポート全体で何が起こったかを要約した簡潔なメールを送信するようにマネージャーから依頼されました。 メールには次の内容を含める必要があります。

- 製品の問題と保証請求の傾向
- 注目すべきエスカレーションまたはディーラーの不満
- 製品またはエンジニアリング部門による調査が必要になる可能性のある新しいパターン
- 今後のステップに関する推奨事項
- サポート エージェントまたはディーラー パートナー向けのトレーニング機会

Microsoft 365 Copilot Chat を使用して初期分析を実行し、メールを作成し、その後に Copilot in Outlook で仕上げる予定です。 Copilot Chat を使用すると、アップロードされた要約、通話メモ、または内部ドキュメントを、プロフェッショナルなトーンの構造化されたレポートに統合できます。 そこから、Copilot in Outlook でメッセージをさらに調整してから、リーダーシップに送信することができます。

一貫性のある分析情報メールを作成することで、Tailwind Traders が事後対応的な問題解決から、データに基づいた事前対応的な改善へと移行するのに役立ちます。 会社の目標は、この移行によってディーラーの満足度を高め、製品のフィードバック ループを強化することです。

#### Copilot Chat を使う  

Web 上の Copilot Chat では、応答モード セレクターを使用して、プロンプトに応答するときに Copilot が使用する時間と推論の量を制御できます。 Copilot が速度と深度のバランスを取るように **[Auto]** (既定のオプション) に設定したままにするか、タスクに応じてより速く、またはより深い応答スタイルを選択することができます。

Copilot Chat が **[作業]** モードで開かれると、応答モード セレクターは表示されません。 **[作業]** モードでは、Copilot はセキュリティで保護された作業コンテキスト クエリ用に最適化されているため、応答の深さを自動的に管理します。 **[Web]** モードに切り替えると、応答モード セレクターが表示され、より高速な応答またはより深い推論を選択できます。 セレクターが有効になった後、**[作業]** モードと **[Web]** モードを切り替えるとセレクターが表示されたままになります。  
<br/>[Copilot in Excel] を使用したことがある場合は、応答コントロール セレクターも含まれていることがわかります。 ただし、そのオプションはチャット セレクターとは異なります。 Copilot Chat では、セレクターによって、要求に関して Copilot がどれだけ深く推論するかが制御されます。 Excel では、セレクターによって、作業を実行する AI モデルが制御されます。 これらのセレクターは似ているかもしれませんが、Copilot の異なる側面を制御するものであり、同じ設定ではありません。

このタスクを完了するには、以下の手順を実行します。

1.  次のリンクを選択して、[**TailwindTraders_SupportCaseSummaries.docx**](https://go.microsoft.com/fwlink/?linkid=2347619) ファイルをダウンロードします ファイルを OneDrive に保存します。

2.  Microsoft Edge ブラウザーで、Microsoft 365 のホーム ページに移動します。

3.  このタスクには、OneDrive にアップロードされたファイルの確認と、その社内ドキュメントからの分析情報の生成が含まれるため、**[Work]** オプションを選択します。 **[Web]** オプションは、公開されている Web サイトやブログなどの外部ソースを検索するため、ここでは適用されません。 Copilot Chat ウィンドウのプロンプト フィールドに **TailwindTraders_SupportCaseSummaries** ファイルを添付します。

4.  次に、添付ファイルにあるサポート ケースの要約を確認し、Outlook で Tailwind Traders の経営幹部に宛てたメールの下書きを作成するよう Microsoft 365 Copilot Chat に依頼します。 主な問題のテーマ、顧客のセンチメント、ディーラーのエスカレーション パターン、推奨される次のアクションを含めてください。 具体的なケース番号は記載しないでください。 

5.  テスト中、私たちは 2 つのシナリオを確認しました。

    - Copilot Chat によってメールが作成され、応答の最後に、その下書きが Outlook に新しいメールとして読み込まれたことが通知される場合がありました。 その後、Outlook でそのメールを開くためのボタンまたはリンクが表示されます。 このような場合は、ボタンまたはリンクを選択して Outlook でメールの内容を開きます。 

    - また、Copilot Chat によってメールの内容は作成されたものの、その内容が Outlook に新しいメールとして読み込まれないこともありました。 このシナリオが発生した場合は、Copilot によって生成されたメールの内容をコピーし、Microsoft 365 のホーム ページに戻り、**[アプリ]** メニューの **[Outlook]** を開いて、新しいメールを追加し、Copilot Chat の応答からメールの内容を貼り付けます。

6.  この時点で、**Outlook on the web** が表示されているはずです。 表示されるのは、Copilot Chat によって生成されたメールか、Outlook で手動で追加し、Copilot Chat の応答メールの内容を貼り付けたメールのいずれかです。 

   > [!IMPORTANT]
   > Copilot Chat でメールを作成し、それを Outlook で開いても、Copilot によってメールが下書きモードで開かれることはありません。 そうではなく、メールの本文内に応答が直接表示されます。 その後、Copilot で変更する特定のテキスト (文、段落、またはメール全体など) を選択する必要があります。

7.  メールを確認した結果、最初の段落に改善の余地があると感じました。 そのためには、メールの最初の段落を選択します (最初の段落全体が選択されるようにカーソルをドラッグします)。 表示される **[Copilot を開く]** (鉛筆) アイコンに注目してください。 このアイコンを選択すると、Copilot ウィンドウが開きます。

8.  Copilot ウィンドウには、プロンプト フィールドと編集オプションのメニューがあります。 最初の段落を選択したので、**[自動書き換え]** オプションを選択します。 修正内容は、Copilot によってメールに直接挿入されないことに注目してください。 そうではなく、修正された内容の下書きを含む Copilot の推敲ウィンドウが表示されます。 また、推敲ウィンドウの下に表示されるメニュー オプションにも注目してください。 次のいずれかを実行できます。
    - 選択したテキストを修正したテキストに置き換えます
    - 選択したテキストの下にテキストを挿入します
    - 書き換えられたテキストを破棄します

    この場合は、**[置換]** オプションを選択します。 選択した最初の段落が、修正されたテキストに置き換えられていることに注目してください。

9.  次に、メールの末尾の「**推奨されるアクション**」セクションの後に新しいセクションを追加しましょう。 メールで、「**推奨されるアクション**」セクションの後にある空白行を選択します。この場所に新しいセクションを配置します。 次に、表示される **[Copilot を開く]** アイコンを選択します。

10.  プロンプト フィールドで、「**リスクと依存関係**」というタイトルのセクションを追加するよう Copilot に依頼します。 このセクションには、対応しないとサポートのパフォーマンスや顧客満足度に悪影響を与える可能性がある項目を説明する短い段落を含める必要があります。

11. その結果を確認します。 Copilot によって新しいセクションが生成され、メール本文内のカーソルを置いた位置に表示されました。 新しい内容に満足したので、Copilot ウィンドウで **[保持する]** を選択します。 

12. 結果を確認します。 Copilot ウィンドウに表示された場所に新しいセクションが挿入されるのではなく、メールの先頭にテキストが挿入されました。 なぜこのようなことが起きたのですか。 さて、この記事の執筆時点では、[保持する] を選択した場合、Copilot によって生成されたテキストは、カーソルの位置やプロンプトで依頼した場所 (たとえば、メールの末尾に新しい段落を追加するように依頼した場合) に正確に挿入されません。 そうではなく、Outlook のメール エディター内の内部プレースホルダーが Copilot に使用されます。現在、このプレースホルダーはメッセージ本文の先頭にあります。

    そのため、Copilot により、視覚的には (どのように見えるかをユーザーに示すために) 生成された段落は依頼した位置に表示されたとしても、**[保持する]** ボタンに関連付けられている実際の挿入ロジックでは、カーソル位置やプロンプトのコンテキストが考慮されません。 その結果、Outlook で現在定義されている既定の挿入ポイント、つまりメッセージ本文の先頭に内容が追加されます。

    この機能が改善されるまでは、挿入されたテキストを選択し、切り取って目的の場所に貼り付ける必要があります。

13.  最後に、メールのトーンを変更するように依頼して、どうなるか見てみましょう。 メール本文内を選択してメール全体を強調表示し、**[Copilot を開く]** アイコンを選択します。 Copilot ウィンドウには、**[トーンの変更]** というメニュー項目があり、**[直接的]**、**[カジュアル]**、**[フォーマル]**、**[詩的に]** という 4 つのオプションが用意されています。 この場合、これらのオプションのいずれかを選択するのではなく、別の内容を求めるプロンプト依頼を送信します。 プロンプト フィールドで、メールのトーンをより経営幹部向けにするよう Copilot に依頼します。

14.  Copilot の推敲ウィンドウの内容をスクロールし、Copilot によってメールがどのように書き換えられ、経営幹部向けのトーンになったかを確認します。 このバージョンが気に入らない場合は、Copilot ウィンドウで **[破棄]** を選択します。 このメールでは、**[置換]** を選択して変更を受け入れます。

15.  時間がある場合は、**[Copilot を開く]** アイコンを選択し、さまざまな Copilot オプションを自由に試してみてください。
