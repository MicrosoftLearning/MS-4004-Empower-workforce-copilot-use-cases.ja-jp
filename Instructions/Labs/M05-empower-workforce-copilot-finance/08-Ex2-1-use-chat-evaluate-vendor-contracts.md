# 演習 2、タスク 1: Microsoft 365 Copilot Chat を使用してベンダー契約を評価する
---
Fabrikam の財務マネージャーは、Adatum Corporation と Contoso, Ltd. の 2 社からのベンダー契約をあなたと共有しました。各契約には、価格設定、納品スケジュール、保証条件、不履行時のペナルティに関する条項が含まれています。 これらの契約の違い、どちらが Fabrikam にとってより大きな財務面または業務面のリスクとなるかを把握する必要があります。 Microsoft 365 Copilot Chat を使用して両方の契約を左右に並べて比較し、主な相違点を特定し、リスク領域を強調し、潜在的なネゴシエーション ポイントを要約し、後で使用できるようにすべてを Word 文書に保存する予定です。

#### Copilot Chat を使う

Web 上の Copilot Chat では、応答モード セレクターを使用して、プロンプトに応答するときに Copilot が使用する時間と推論の量を制御できます。 Copilot が速度と深度のバランスを取るように **[Auto]** (既定のオプション) に設定したままにするか、タスクに応じてより速く、またはより深い応答スタイルを選択することができます。

Copilot Chat が **[作業]** モードで開かれると、応答モード セレクターは表示されません。 **[作業]** モードでは、Copilot はセキュリティで保護された作業コンテキスト クエリ用に最適化されているため、応答の深さを自動的に管理します。 **[Web]** モードに切り替えると、応答モード セレクターが表示され、より高速な応答またはより深い推論を選択できます。 セレクターが有効になった後、**[作業]** モードと **[Web]** モードを切り替えるとセレクターが表示されたままになります。

**[Copilot in Excel]** を使用したことがある場合は、応答コントロール セレクターも含まれていることがわかります。 ただし、そのオプションはチャット セレクターとは異なります。 Copilot Chat では、セレクターによって、要求に関して Copilot がどれだけ深く推論するかが制御されます。 Excel では、セレクターによって、作業を実行する AI モデルが制御されます。 これらのセレクターは似ているかもしれませんが、Copilot の異なる側面を制御するものであり、同じ設定ではありません。

このタスクでは、既定の **[Auto]** セレクター モードが使用されます。

このタスクを完了するには、次の手順を実行します。

1.  次のリンクを選択して 2 つのベンダー契約をダウンロードし、テナント内の Copilot で使用するためにファイルを OneDrive アカウントに保存します。
    - [**Adatum Corp smart sensor contract.docx**](https://go.microsoft.com/fwlink/?linkid=2347801)
    - [**Contoso Ltd smart sensor contract.docx**](https://go.microsoft.com/fwlink/?linkid=2347516)

2.  必要に応じて、Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインします。  
    
3.  **[作業] オプションを選択します。これは**組織内のコンテンツ (OneDrive、SharePoint、Teams、Outlook) の検索と分析に使用されます。 プロンプト フィールドに、手順 1 でダウンロードした各ベンダー契約を添付します。

4.  以下のプロンプトを送信して、2 つの契約を比較するように Copilot に依頼します。

   > [!NOTE]
   > この最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素が組み込まれている場合に、効果的なプロンプトがどのようになるかわかるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて学習者が記述する必要がありますが、その際は、このプロンプトを手本として参考にできます。

   **私は Fabrikam, Inc. の財務アナリストです。コスト最適化とリスク削減のイニシアティブの一環として、2 つのベンダー契約を評価しています。これらの契約は、新しいスマート センサー製品ラインをサポートするコンポーネントに関するものです。Adatum Corporation と Contoso, Ltd. の契約を比較し、Fabrikam のコスト、サプライヤー リスク、長期的な財務実績に影響を与える可能性のある主な相違点を特定してください。添付した 2 つのベンダー契約を参考資料として使用してください。価格設定**、**納期**、**保証、賠償責任、または違約金に関する条項に焦点を当ててください。2 つの契約の主な相違点、Fabrikam にとってより高い財務リスクと運用リスクをもたらす契約、Fabrikam が条件改善のために提示できる 2 点から 3 点のネゴシエーション ポイントを、箇条書きでわかりやすくまとめてください。**

5.  Copilot によって生成された比較を確認します。 応答の最後に、**[Pages で編集]** アイコンを選択します。

6.  **Pages** での編集中に、Copilot によって Pages フォームと共に [Copilot Chat] ペインが表示されることに注目してください。 **Pages** フォームで **[作成]** ボタンを選択し、ドロップダウン メニューから **[文書]** を選択します。 表示されるダイアログ ボックスで、**[Word を開く]** を選択すると、文書が **Word for the web** で開きます。

7.  **Word for the web** では、チャットの最初と最後に表示された不要なチャット内容を含め、応答全体が Copilot によって文書にコピーされます。 貼り付けられたテキストに不要なものがあれば削除します。メニュー バーの上に表示されているファイル名を選択し、ファイル名を **Smart Sensor contract comparison** に変更します。 後の手順でさらに情報を追加する予定なので、文書は開いたままにしてください。

8.  Word 文書で、**[ホーム]** タブのリボンにある **[Copilot]** オプションを選択します。 プロンプト フィールドのプラス (+) 記号の横に **[Copilot で編集]** アイコンが表示されていることを確認します。 表示されていない場合は、プラス記号を選択し、ドロップダウン メニューから **[Copilot で編集]** を選択します。 プロンプト フィールドにアイコンが表示されます。

9.  次に、2 つの契約を比較する「要約」セクションを文書に追加するよう Copilot に依頼する必要があります。 ただし、その前に、新しいセクションを表示する文書内の場所にカーソルを置く必要があります。 この「要約」を文書の先頭に表示する場合は、最初のセクションの上に空白行を挿入し、その行にカーソルを置きます。

10.  次に、2 つのベンダー契約の要約分析が記載された「要約」セクションを文書に追加するよう Copilot に依頼します。 このセクションには、どちらの契約が Fabrikam にとってより有利な条件であるか、そしてその理由を示す必要があります。 また、このセクションでは、Fabrikam がネゴシエーションの労力を集中させるべき点についても示す必要があります。

11.  Copilot によって文書に追加されたセクションを確認します。 これは良い出発点ではありますが、改善の余地があります。 実際に財務アナリストがサプライヤー契約を確認する場合、分析に含める可能性のある領域は多数あります。 一般的な領域は次のとおりです。

        - **契約終了と退出条項**。 通常、企業は、通知期間や早期終了の条件、契約終了に伴うペナルティや費用、在庫や支払いなどの未履行義務が終了時にどのように扱われるかに重点を置きます。

        - **数量の柔軟性とスケーラビリティ**。 契約には、ペナルティなしで発注量を増減できる規定、発注量の増加に応じて価格がどのように変動するか、需要の変化に応じて最小発注数量を調整できるかどうかが含まれるのが一般的です。

        - **変更管理と修正**。 通常、アナリストは、契約の修正や仕様変更がどのように扱われるか、テクノロジや規制要件の変更を管理する正式なプロセスがあるかどうかを確認したいと考えます。

        - **サプライヤーのパフォーマンスと監視**。 通常、企業は、サプライヤー契約の中で主要業績評価指標 (KPI) やサービス レベル アグリーメント (SLA) を定義し、サプライヤーのパフォーマンスが継続的に低かった場合の救済措置、定期的なパフォーマンス レビューや監査のプロセスがあるかどうかを確認します。

        - **コンプライアンスと規制の要件**。 通常、アナリストは、契約が特定の業界標準や規制への準拠を求めているかどうか、準拠を確認するための監査権があるかどうかを確認したいと考えます。

        - **紛争解決**。 通常、企業は、仲裁、調停、管轄など、紛争を解決するためのしくみが契約でどのように定められているかを確認したいと考えます。

        - **サプライヤーの財務状況と安定性**。 サプライヤーによっては、特定の財務指標や保険レベルの維持を求める条項、サプライヤーが買収された場合や重大な財務上の困難に陥った場合にどうなるかを契約に含めたいと企業が考えることもあります。

        複数の領域にわたる契約を一度に比較するよう Copilot に依頼するのではなく、自分にとって関心のある領域を 1 つだけ選択します。 Copilot で、選択したトピックを分析するセクションを追加します。 依頼する前に、新しいセクションを表示する文書内の場所にカーソルを置きます。 次に、その点に特化して契約を比較する新しいセクションを文書に追加するよう Copilot に依頼します。 多くの場合、1 つのプロンプトで複数の依頼を送信して Copilot に負担をかけるよりも、1 つの側面に焦点を当てることで、より詳細かつ正確な結果が得られます。

12.  Copilot によって文書に追加された新しいセクションを確認します。 次は、2 つ目のトピックを分析するセクションを追加するよう Copilot に依頼します。 依頼する前に、新しいセクションを表示する文書内の場所にカーソルを置きます。 自分にとって関心のある 2 つ目のトピックについて契約を比較するセクションを文書に追加するよう Copilot に依頼します。

13.  Copilot によって文書に追加された新しいセクションを確認します。 次のタスクでは、この文書を基に PowerPoint プレゼンテーションを作成する予定なので、必要に応じて自由に改良を加えます。 たとえば、時間があれば、先ほどの一覧にある別のトピックを比較するセクションを追加することもできます。 また、Copilot ペインの最後に表示された Copilot の提案プロンプトを確認し、自分にとって関心のある依頼を送信することもできます。
