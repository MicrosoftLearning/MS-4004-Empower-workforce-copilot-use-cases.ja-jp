# 演習 2、タスク 1: Microsoft 365 Copilot Chat を使用してベンダー契約を評価する
---
Fabrikam の財務マネージャーは、Adatum Corporation と Contoso, Ltd. の 2 社からのベンダー契約をあなたと共有しました。各契約には、価格設定、納品スケジュール、保証条件、不履行時のペナルティに関する条項が含まれています。 これらの契約の違い、どちらが Fabrikam にとってより大きな財務面または業務面のリスクとなるかを把握する必要があります。 Microsoft 365 Copilot Chat を使用して両方の契約を左右に並べて比較し、主な相違点を特定し、リスク領域を強調し、潜在的なネゴシエーション ポイントを要約し、後で使用できるようにすべてを Word 文書に保存する予定です。

このタスクを完了するには、以下の手順を実行します。

1.  次のリンクを選択して 2 つのベンダー契約をダウンロードし、テナント内の Copilot で使用するためにファイルを OneDrive アカウントに保存します。
    - [**Adatum Corp smart sensor contract.docx**](https://go.microsoft.com/fwlink/?linkid=2347801)
    - [**Contoso Ltd smart sensor contract.docx**](https://go.microsoft.com/fwlink/?linkid=2347516)

2.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** に移動し、**[作業] オプションを選択します。** このオプションは、内部組織コンテンツ (OneDrive、SharePoint、Teams、Outlook) の検索と分析に使用されます。

3.  両方の契約を Copilot プロンプトに添付し、2 つの契約を比較するよう Copilot に依頼します。
  
   > [!NOTE]
   > 最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素を取り入れた効果的なプロンプトがどのようなものかを確認できるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて自分で記述する必要がありますが、その際には、このプロンプトを手本として参考にできます。
    
   **私は Fabrikam, Inc. の財務アナリストです。コスト最適化とリスク削減のイニシアティブの一環として、2 つのベンダー契約を評価しています。これらの契約は、新しいスマート センサー製品ラインをサポートするコンポーネントに関するものです。Adatum Corporation と Contoso, Ltd. の契約を比較し、Fabrikam のコスト、サプライヤー リスク、長期的な財務実績に影響を与える可能性のある主な相違点を特定してください。添付した 2 つのベンダー契約を参考資料として使用してください。価格設定**、**納期**、**保証、賠償責任、または違約金に関する条項に焦点を当ててください。2 つの契約の主な相違点、Fabrikam にとってより高い財務リスクと運用リスクをもたらす契約、Fabrikam が条件改善のために提示できる 2 点から 3 点のネゴシエーション ポイントを、箇条書きでわかりやすくまとめてください。**

4.  Copilot によって生成された比較を確認します。 Word 文書に保存する場合は、応答後に表示される **[コピー]** アイコンを選択します。

5.  Microsoft Edge ブラウザーで新しいタブを開き、**Microsoft 365** のホーム ページに移動し、ナビゲーション ウィンドウで **[アプリ]** を選択して、[アプリ] メニューから **[Word]** を選択します。
   
6.  **Word for the web** の Word で空白の文書を開き、応答を貼り付けます。 チャット セッションからコピーして貼り付けたテキストに余分な部分があれば削除します。 「**Smart Sensor contract comparison**」と名前を付けて文書を保存します。 後の手順でさらに情報を追加する予定なので、文書は開いたままにしてください。

7. Copilot Chat セッションが表示されているブラウザー タブに戻ります。 Fabrikam にとってより有利な契約とその理由を Copilot に尋ねます。 また、Fabrikam がネゴシエーションの取り組みの焦点をどこに置けばよいかについても尋ねます。

8.  応答をコピーし、**Smart Sensor contract comparison** 文書の末尾に貼り付けます。 チャット セッションからコピーして貼り付けたテキストに余分な部分 (通常はコピーしたテキストの先頭と末尾) があれば削除します。

9.  Copilot Chat セッションに戻ります。 実際に財務アナリストがサプライヤー契約を確認する場合、分析に含める可能性のある領域は多数あります。 一般的な領域は次のとおりです。

    - **契約終了と退出条項**。 通常、企業は、通知期間や早期終了の条件、契約終了に伴うペナルティや費用、在庫や支払いなどの未履行義務が終了時にどのように扱われるかに重点を置きます。

    - **数量の柔軟性とスケーラビリティ**。 契約には、ペナルティなしで発注量を増減できる規定、発注量の増加に応じて価格がどのように変動するか、需要の変化に応じて最小発注数量を調整できるかどうかが含まれるのが一般的です。

    - **変更管理と修正**。 通常、アナリストは、契約の修正や仕様変更がどのように扱われるか、テクノロジや規制要件の変更を管理する正式なプロセスがあるかどうかを確認したいと考えます。

    - **サプライヤーのパフォーマンスと監視**。 通常、企業は、サプライヤー契約の中で主要業績評価指標 (KPI) やサービス レベル アグリーメント (SLA) を定義し、サプライヤーのパフォーマンスが継続的に低かった場合の救済措置、定期的なパフォーマンス レビューや監査のプロセスがあるかどうかを確認します。

    - **コンプライアンスと規制の要件**。 通常、アナリストは、契約が特定の業界標準や規制への準拠を求めているかどうか、準拠を確認するための監査権があるかどうかを確認したいと考えます。

    - **紛争解決**。 通常、企業は、仲裁、調停、管轄など、紛争を解決するためのしくみが契約でどのように定められているかを確認したいと考えます。

    - **サプライヤーの財務状況と安定性**。 サプライヤーによっては、特定の財務指標や保険レベルの維持を求める条項、サプライヤーが買収された場合や重大な財務上の困難に陥った場合にどうなるかを契約に含めたいと企業が考えることもあります。

10.  複数の領域について一度に契約を比較するよう Copilot に依頼するのではなく、自分が関心のある領域を 1 つだけ選択し、その点に絞って契約を比較するよう Copilot に依頼してください。 多くの場合、1 つのプロンプトで複数の依頼を送信して Copilot に負担をかけるよりも、1 つの側面に焦点を当てることで、より詳細かつ正確な結果が得られます。

11.  応答をコピーし、**Smart Sensor contract comparison** 文書の末尾に貼り付けます。 チャット セッションからコピーして貼り付けたテキストに余分な部分があれば削除します。

12.  前の手順を繰り返し、関心のある他の 1 つの領域の契約を比較するよう Copilot に依頼します。

13.  応答をコピーし、**Smart Sensor contract comparison** 文書の末尾に貼り付けます。 チャット セッションからコピーして貼り付けたテキストに余分な部分があれば削除します。

14.  時間に余裕があれば、他の領域も比較してみてください。 最終的な変更内容は、必ず **Smart Sensor contract comparison** 文書に保存してください。 次のタスクでは、この文書に基づいて PowerPoint プレゼンテーションを作成します。
