# 演習 2、タスク 1: Copilot in Loop を使用して建設のマイルストーンを追跡する
---
あなたは Contoso の地域物流センター拡張のオペレーション リーダーであり、プロジェクトをスケジュールどおりに進め、すべてのチームの連携を確保する責任があります。 拡張には、敷地の準備、建設、検査、機器の設置、最終引き渡しなど複数のフェーズが含まれ、それぞれに独自の依存関係と期限があります。

流動的な要素と利害関係者 (建設作業員、物流、安全、IT、リーダーシップ) が非常に多いため、進行状況を追跡し、責任を割り当てるための明確で視覚的な方法が不可欠です。 Copilot によって強化された Microsoft Loop は、すべてのマイルストーンを整理し、監視するための中心的なハブです。 プロジェクト計画の Loop ワークスペース内で、次のページを作成する予定です。

- 1-マイルストーンとタイムライン
- 2-RAID ログ (リスク、前提、問題、依存関係)
- 3-RACI マトリックス (役割と責任)
- 4-決定ログと変更ログ

この種の一般的な建設プロジェクトには、安全トレーニング ロールアウト計画、在庫移動ウェーブ計画、利害関係者のコミュニケーション計画、運用開始 Runbook など、Loop で追跡できる他の多くのタスクがあります。 ただし、時間の都合上、このタスクでは上記の 4 ページのみを作成します。

このタスクを完了するには、以下の手順を実行します。

1.  Microsoft Edge ブラウザーで、**Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインし、ナビゲーション ウィンドウで **[アプリ]** を選択して、**[アプリ]** メニューから **[Loop]** を選択します。

2.  **Loop for the web** で、「**物流センターの拡張 - プロジェクト計画**」というタイトルの新しいワークスペースを作成します。

3.  あなたは、この Loop ワークスペースの下に、交渉戦略用、リスク軽減策のアイデア用、コミュニケーション フレーミング用という 3 つの新しいページを作成する予定です。 最初のページの現在のタイトルを「**無題**」から「**1-マイルストーンとタイムライン**」に変更します。

4.  Copilot ペインを開きます。 次のプロンプトを送信して、物流センター拡張の主要なマイルストーンとそれに関連するタイムラインの表を生成するよう Copilot に依頼します。

   > [!NOTE]
   > 最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素を取り入れた効果的なプロンプトがどのようなものかを確認できるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて自分で記述する必要がありますが、その際には、このプロンプトを手本として参考にできます。
    
   **私は、ノースダコタ州ファーゴにある Contoso 社の物流センター拡張プロジェクトを統括するオペレーション リーダーです。このプロジェクトには、複数の建設フェーズ、安全対策の更新、ベンダーとの調整、在庫移動が含まれます。この拡張における主要なマイルストーンをすべて網羅した詳細なプロジェクト計画を作成してください。計画は、タスク名、バケット/フェーズ、開始日、期限、依存関係、所有者 (役割)、チェックリスト項目の列がある表にまとめてください。"現場準備"、"建設"、"施設システム"、"検査"、"稼働準備" などの論理バケットを使用してください。プロジェクトは来週月曜日に開始され、24 週間続くと想定してください。編集可能な Loop テーブルとして計画を返してください。**

5.  テーブルの結果を確認します。 テーブルを Loop ページに直接挿入するかどうかを Loop から確認される場合があります。 テスト中、テーブルを Loop ページに挿入するように指示するボタンや、プロンプト候補は Copilot によって表示されませんでした。 それを実行するように依頼するプロンプトを入力すると、同じテーブルが再生成されましたが、挿入されませんでした。 Copilot in Loop はまだ開発中であるため、テーブルを Loop ページに挿入する最も簡単な方法は、テーブルの下に表示される **[コピー]** アイコンを選択することです。 コピーした結果を「**1-マイルストーンとタイムライン**」ページに貼り付けます。 Copilot ペインの会話から、テーブルと共にコピーして貼り付けた余分なテキスト (通常は貼り付けた内容の先頭と末尾) を削除します。

6.  すべてのテーブル列を表示するには、横方向にスクロールします。 実際の状況では、オペレーション リーダーは、日付や所有者の役割を編集し、チェックリスト項目を追加または削除し、必要に応じてタスクを並べ替えることで、運用上の推論を実行します。 この演習では、テーブルの結果をそのままにしておくことができます。

7.  次に、"**物流センターの拡張 - プロジェクト計画**" ワークスペースの下に新しいページを追加します。 ページのタイトルを「**無題**」から「**2-RAID ログ (リスク/前提/問題/依存関係)**」に変更します。

8.  Copilot ペインを開きます。 厳しい日程の中で建設、安全、在庫移動、ベンダー間の調整を容易にするために、影響や可能性のスコア付けやオペレーション部門の役割に連動した軽減策の所有者など、今後 24 週間の主要な項目を取り込む RAID ログ テーブルを作成するよう Copilot に依頼します。 この Loop テーブルには、種類、タイトル、説明、影響度 (高/中/低)、可能性 (高/中/低)、所有者 (役割)、目標日、軽減策/アクション、状態の列を含める必要があります。 テーブルの下に、簡潔な段落でクリティカル パスのリスクを定義してください。

9.  表の結果と、Copilot によって生成されたクリティカル パスのリスクを示す段落を確認します。 問題がなければ、内容をコピーして「**2-RAID ログ**」ページに貼り付け、余分なテキストを削除します。

10.  次に、"**物流センターの拡張 - プロジェクト計画**" ワークスペースの下に 3 ページ目を追加します。 ページのタイトルを「**無題**」から「**3-RACI マトリックス (役割と責任)**」に変更します。

11.  Copilot ペインを開きます。 オペレーション チームがタスク所有者を割り当て、コミュニケーション フローを明確する作業を進めやすいように、サイトの準備、基礎、フレーミング、電気、スプリンクラーのテスト、ドックのアップグレード、検査、機器の設置、稼働準備などのマイルストーンの RACI マトリックスを作成するよう Copilot に依頼します。 これを行うために、行 = マイルストーン、列 = 役割 (オペレーション リーダー、建設リーダー、安全マネージャー、物流コーディネーター、財務、IT、ベンダー PM)、値 = R/A/C/I という Loop テーブルを作成してください。

12.  テーブルの結果を確認します。 問題がなければ、テーブルをコピーして「**3-RACI マトリックス**」ページに貼り付け、余分なテキストを削除します。

13.  "**物流センターの拡張 - プロジェクト計画**" ワークスペースの下に最終ページを追加します。 ページのタイトルを「**無題**」から「**4-決定ログと変更ログ**」に変更します。

14.  Copilot ペインを開きます。 このページには、決定ログと変更ログの 2 つのテーブルが含まれています。 両方のテーブルを一度に生成するよう Copilot に依頼するのではなく、個別のプロンプト依頼を送信する予定です。 オペレーション リーダーが週末のシフト、ベンダーの時間、PPE 予算の今後の承認を追跡できるようにするために、決定、依頼者、期日、検討されたオプション、最終決定、根拠、所有者、フォローアップ タスクの列を含む決定ログ テーブルを作成するよう Copilot に依頼します。

15.  テーブルの結果を確認します。 問題がなければ、決定ログ テーブルをコピーして「**4-決定ログと変更ログ**」ページに貼り付け、余分なテキストを削除します。

16.  次に、検査日、ルーティング、稼働準備の潜在的な変更を追跡する変更ログ テーブルを作成するよう Copilot に依頼します。 このテーブルには、変更依頼、カテゴリ (スケジュール/スコープ/コスト/品質)、説明、影響の要約、必要な承認、状態、所有者、有効日の列を含める必要があります。

17.  テーブルの結果を確認します。 問題がなければ、変更ログ テーブルをコピーして「**4-決定ログと変更ログ**」ページに貼り付け、余分なテキストを削除します。

18.  以降のタスクで、このタスクで作成した Loop ワークスペースへのリンクを含むメールを、オペレーション部門の同僚宛てに作成します。 リンクをコピーできるように、"**物流センターの拡張 - プロジェクト計画**" ワークスペースを開いたままにしておきます。
