# 演習 1、タスク 4: Microsoft 365 Copilot Chat を使用してベンダー エンゲージメントの準備を行う
---
ここまで大変な 1 日でした。 Copilot の助けを借りて、Whiteboard でブレーンストーミングし、Word で調査し、PowerPoint でエグゼクティブ プレゼンテーションを作成しました。 午後も遅くなりましたが、一日を終える前に、意思決定プロセスを完了し、ベンダー エンゲージメントの準備を行う必要があります。 

この最後のタスクでは、これまでの作業を統合して以下の実行可能な出力を作成します。

- ボイラーの修理、ボイラーの交換、炉への切り替えを比較する明確な意思決定マトリックス。

- HVAC ベンダーに送信する提案依頼書 (RFP) の下書き。

- リーダーシップが次のステップを承認する前に確認できる経営幹部向けの概要。

このプロセスを支援するために、Microsoft 365 Copilot Chat を使用する予定です。 Chat は、以前の成果物 (Whiteboard のアイデア、Word レポート、PowerPoint プレゼンテーション) からコンテキストをプルし、それらを構造化された成果物に統合するのに役立ちます。これらはすべて、Microsoft 365 エコシステムを離れることなく行うことができます。

#### Copilot Chat を使う  

Web 上の Copilot Chat では、応答モード セレクターを使用して、プロンプトに応答するときに Copilot が使用する時間と推論の量を制御できます。 Copilot が速度と深度のバランスを取るように **[Auto]** (既定のオプション) に設定したままにするか、タスクに応じてより速く、またはより深い応答スタイルを選択することができます。

Copilot Chat が **[作業]** モードで開かれると、応答モード セレクターは表示されません。 **[作業]** モードでは、Copilot はセキュリティで保護された作業コンテキスト クエリ用に最適化されているため、応答の深さを自動的に管理します。 **[Web]** モードに切り替えると、応答モード セレクターが表示され、より高速な応答またはより深い推論を選択できます。 セレクターが有効になった後、**[作業]** モードと **[Web]** モードを切り替えるとセレクターが表示されたままになります。  

[Copilot in Excel] を使用したことがある場合は、応答コントロール セレクターも含まれていることがわかります。 ただし、そのオプションはチャット セレクターとは異なります。 Copilot Chat では、セレクターによって、要求に関して Copilot がどれだけ深く推論するかが制御されます。 Excel では、セレクターによって、作業を実行する AI モデルが制御されます。 これらのセレクターは似ているかもしれませんが、Copilot の異なる側面を制御するものであり、同じ設定ではありません。

このタスクを完了するには、次の手順を実行します。

1.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページにサインインします。 このタスクには、OneDrive にアップロードされたファイルの確認と、その社内ドキュメントからの分析情報の生成が含まれるため、**[Work]** オプションを選択します。 **[Web]** オプションは、公開されている Web サイトやブログなどの外部ソースを検索するため、ここでは適用されません。

2.  まず、Copilot にベンダー エンゲージメントの準備をさせる必要があります。 具体的には、Copilot Chat を使用して、この暖房システム プロジェクトの HVAC ベンダーを選択するために使用する必要がある決定基準をまとめます。 あなたは、Copilot でこの情報を使用して、意思決定マトリックスを事前設定する予定です。 まず、タスク 2 で作成した **Heating System Comparison** レポートをこのプロンプトに添付します。

3.  次に、このレポートの主要な意思決定要因を要約するように Copilot Chat に依頼します。 決定基準には、コスト、エネルギー効率、ダウンタイムの影響、長期的なメンテナンスに関する考慮事項を含める必要があります。

4.  決定基準を確認し、必要に応じて改善します。 Copilot が表示するプロンプト候補を確認し、条件の改善に役立つと思われるプロンプトを送信します。

5.  意思決定基準が設定されたので、3 つの選択肢 (ボイラーを修理する、ボイラーを交換する、暖房炉に切り替える) を比較する意思決定マトリックスを生成するように Copilot に依頼します。 コスト、設置時間、エネルギー効率、予想される寿命、リスク レベルの列を含めます。 Copilot には、意思決定マトリックスの列を事前設定するときは以前に作成した意思決定基準を使用するように指示し、リーダーシップの関心、コスト、エネルギー効率、ダウンタイム、メンテナンスを基準に比較されるようにする必要があります。

6.  マトリックスの正確性と完全性を確認します。 好調なスタートですが、重要な情報がいくつか欠けていると感じています。 保証条件とコンプライアンス要件の列を追加するように Copilot に依頼します。

7.  結果を確認します。 すべての意思決定基準が設定されたことを確認したので、次は意思決定マトリックスをより実用的なものにする必要があります。 基準に対して各オプションを評価またはスコア付けするように Copilot に依頼します。 Copilot は、"リスク レベル" を列として一覧表示するのではなく、オプションごとに高、中、または低のリスク評価を割り当てる必要があります。 高、中、または低の評価を追加すると、マトリックスは、オプションの比較にすぐに使用できます。

9.  意思決定マトリックスを確認します。 Copilot が表示するプロンプト候補を確認し、意思決定マトリックスの改善に役立つと思われるものを送信します。

8.  意思決定マトリックスの更新が完了したら、それを意思決定ツールとして、そして RFP とエグゼクティブ サマリーの基礎として使用できます。 タイトル、概要、意思決定マトリックス テーブル、ベンダーの選択にマトリックスを使用する方法を説明する使用ガイダンスの段落を含むダウンロード可能な Word レポートとして意思決定マトリックスを書式設定するように Copilot に依頼します。 使用ガイダンスの段落の目的は、リーダーシップがマトリックスの解釈方法を理解できるようにすることです。

10. Copilot からダウンロード リンクが提供されたら、意思決定マトリックスの文書をダウンロードします。

11. これで、Copilot Chat で提案依頼書 (RFP) を生成する準備ができました。 Copilot Chat のプロンプトに、ダウンロードした意思決定マトリックスの文書を添付し、添付ファイルの意思決定基準とマトリックスを使用して HVAC ベンダー向けの RFP の下書きを作成するように Copilot に指示します。 意思決定マトリックスは、Copilot が評価基準に合わせて RFP を調整するのに役立ちます。 RFP には、プロジェクト スコープ、タイムライン、評価基準を含める必要があります。

12. Copilot が表示するプロンプト候補を確認し、RFP の改善に役立つと思われるプロンプトを送信します。

13. RFP をダウンロード可能な Word 文書に書式設定するように Copilot に依頼します。 Copilot からダウンロード リンクを提供されたら、RFP をダウンロードします。

14. これでプロセスは終了です。 この時点で、Copilot を使用して、推奨事項の基礎として意思決定マトリックスを参照するエグゼクティブ サマリーを作成します。 Copilot に次の依頼を行います。

    - 事情 (故障したボイラーと対処が必要な理由) を提供する 1 ページのエグゼクティブ サマリーを作成する

    - 使用可能なオプション (ボイラーの修理、交換、またはボイラーから炉への切り替え) について説明する

    - 意思決定マトリックスからハイライト (費用の範囲、エネルギー効率、リスク レベルなど) を引き出す

    - 推奨されるオプションを提案する

    - 意思決定マトリックスに基づいて次の手順 (RFP の承認、予算の割り当てなど) を推奨する

    要約は、意思決定にすぐに使用できる必要があります。つまり、レポートやマトリックスの完全に調べなくても状況を素早く理解し、情報に基づいた選択を行うために必要なすべての情報をリーダーシップに提供する必要があります。

15.  Copilot が表示するプロンプト候補を確認し、エグゼクティブ サマリーの改善に役立つと思われるプロンプトを送信します。

16.  エグゼクティブ サマリーをダウンロード可能な Word 文書に書式設定するように Copilot に依頼します。 Copilot からダウンロード リンクを提供されたら、文書をダウンロードします。
