# 演習 2:人事セルフサービス エージェントを使用して従業員を支援する
---
最近は、従業員サポートの強化、コミュニケーションの合理化、管理負荷の軽減のために Microsoft 365 Copilot を使用する人事部門が増えています。 カスタム構築の Copilot エージェントと Microsoft の事前構築済みエージェントを組み合わせると、人事チームは、会社のポリシーとの一貫性を維持しながら、従業員の問い合わせに対してより迅速かつ正確な応答を提供できます。

たとえば、人事部門は、従業員が人事スタッフを待たずに、福利厚生、昇進、転勤、職場のポリシーに関する情報に即座にアクセスできるようにするカスタム エージェントを構築できます。 カスタム構築のエージェント スキャンを使用すると、サービス品質と運用効率の両方が向上します。

一方、リサーチ ツールなどの事前構築済みの Copilot エージェントを使用すると、従業員や人事担当者は、外部の分析情報を収集して合成し、情報に基づいた意思決定を行うことができます。 これらのツールを組み合わせて、Microsoft 365 のインテリジェント エージェントがどのようにして人事部門の影響を拡大し、つながりがより強いデータ主導の職場を育成できるかを例示します。

> [!TIP]
> このモジュールの「概要」ユニットでは、効果的なプロンプトの 4 つの重要な要素である目標、コンテキスト、ソース、期待される回答について説明しました。 この演習でプロンプトを作成する際は、これらの要素に留意してください。

### シナリオ

この演習では、2 つの役割を担います。1 つ目は、新しいエージェントの人事セルフサービス エージェントと Viva Engage アナウンスの両方を作成する人事アナリストとしての役割で、2 つ目は、最近の昇進後に新しい都市に転勤する Adatum 従業員としての役割です。 

この従業員は、人事セルフサービス エージェントを使用して、会社の転勤ポリシーを含む会社のポリシーに関する回答を得ます。 次に、転勤に備えて、リサーチ ツール エージェントを使用して、従業員の新しい年の学校を調べます。

この演習では、Copilot Chat がどのようにして会社のデータと Web ソースを接続し、社内業務と個人の生産性の両方をサポートするかを例示します。
