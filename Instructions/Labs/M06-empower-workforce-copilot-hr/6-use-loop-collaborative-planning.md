# 演習 1、タスク 4: Copilot in Loop を使用して共同のマネージャー育成計画を促進する
---
Contoso は、マネージャーのパフォーマンスとチームの健全性に関する年次レビューを完了したところです。 このレビュー後、Contoso の最高人事責任者である Holly Dickson からあなたに対して、マネージャーがデータに基づく分析情報を具体的な育成アクションに変換できるよう支援する共同計画セッションを主導するよう求められました。 Holly は、すべてのマネージャーが明確で実行可能なフィードバックを受け取り、成長できる領域に取り組む計画を立てられるようにしたいと考えています。

> [!NOTE]
> このタスクで使用されるマネージャー データは架空のものですが、あなたの目標は、実際の組織で人事部門が結果の解釈、目標の設定、進行状況の追跡をマネージャーに対してどのように指導するかをシミュレートすることです。

このプロセスを整理し、促進するために、Microsoft Loop を使用する予定です。 マネージャー育成計画の中心ハブとして機能する共同 Loop ワークスペースを作成する必要があります。 Copilot in Loop は、実際のマネージャーと一緒に作業する場合と同じように、調査結果を要約し、アクション ステップを生成し、ディスカッションを促進するのに役立ちます。

このタスクを完了するには、以下の手順を実行します。

1.  Microsoft Edge ブラウザーで、**Microsoft 365** のホーム ページに移動し、ナビゲーション ウィンドウで **[アプリ]** を選択し、**[アプリ]** メニューから **[Loop]** を選択します。 一覧に **Loop** が表示されない場合は、**[すべてのアプリ]** を選択します。 **[アプリ]** ウィンドウのアプリの上部セクションで、**[すべてのアプリ]** を選択します。 **[すべてのアプリ]** ウィンドウで下にスクロールし、**[Loop]** を選択します。

2.  **Loop for the web** で、「**Contoso マネージャー育成計画**」というタイトルの新しいワークスペースを作成します。

3.  Manager Performance PDF ファイルには、以前のタスクで分析した 15 人の Contoso マネージャーに関する分析情報が含まれていますが、このトレーニングでは 15 人全員の調査結果を要約する時間が足りません。 この演習が実際のシナリオでどのように機能するかを理解しつつ、時間がかかりすぎないようにするために、このタスクでは、マネージャーの Jacob Hancock と Kerry Allen の 2 名にのみ焦点を当てます。  
    
    各マネージャーの Loop ページを作成します。 まず、最初のページの現在のタイトルを「**無題**」から「**Jacob Hancock**」に変更します。 次に、この Loop ワークスペースの下に **Kerry Allen** 用の別のページを作成します。

4.  「**Jacob Hancock**」ページを選択し、Copilot ペインを開きます。

5.  Copilot プロンプト フィールドに、タスク 2 の最後に作成して OneDrive に保存した Manager Performance PDF ファイルを添付します。 これを行うには、プロンプト フィールドで **[参照ファイル]** (+ 記号) アイコンを選択し、タスク 2 で作成した PDF ファイルを選択します。 

   > [!TIP]
   > 表示されるファイル メニューに PDF ファイルが表示されない場合は、プロンプト フィールドに「**/Contoso**」と入力して、OneDrive 内の **Contoso** で始まるすべてのファイルを表示します。 一覧には、タスク 2 で作成した PDF ファイルが含まれているはずです。 このファイルを選択します。

6.  添付のレポートに基づいて、Jacob Hancock の主な強みと育成領域を要約するよう Copilot に依頼します。 また、チームのエンゲージメントを向上させ、離職率を下げるために Jacob Hancock が実行できる 3 つの実行可能なステップを提案するよう Copilot に依頼します。

7.  結果を確認します。 結果の後に表示される **[コピー]** アイコンを選択し、コピーした内容を Jacob の Loop ページに貼り付けます **(Ctrl + V)**。 ページに貼り付けられた内容の先頭と末尾に表示される会話テキストを削除します。

8.  **Kerry Allen** の Loop ページを選択します。 何が起きたかに注目してください。Copilot ペインが消えました。 次に、プロンプト候補を下にスクロールして、**[チャット履歴]** ボタンを選択します。 このドキュメントには以前の Copilot チャットがないことを示すメッセージが表示されます。 つまり、Loop ページを切り替えると、別のページに関連付けられたチャット履歴にはアクセスできません。
    
   > [!NOTE]
   > Microsoft Loop では、Copilot ペインは状況依存であり、現在開いているページ内でのみ機能します。 アクセスするプロンプト、応答、チャット履歴はそのページ固有のものであり、Copilot では、ワークスペース内の他のページの会話は保持または表示されません。 この設計により、分析とコラボレーションが常に集中して整理された状態になり、各ページが独自の Copilot インサイトとディスカッションの個別のワークスペースとして機能します。

9.  **Jacob Hancock** の Loop ページをもう一度選択します。 Copilot に新しいペインが表示され、Jacob に関連する以前の会話が表示されない点に注目してください。 Loop ページに戻ったら、そのページで新しい Copilot チャット セッションを開始するか、既存のセッションに戻ることができます。 

    以前の会話に戻る方法を確認するには、プロンプト候補を下にスクロールし、**[チャット履歴]** ボタンを選択します。 この Loop ページには、以前の各チャット セッションの行が表示されます。 この場合、チャット セッションは 1 つだけなので、その行を選択してその会話を開きます。 

10.  先ほどは、Copilot in Loop が Loop ページ内のチャット セッションをどのように処理するかを確認するために、前の 2 つの手順を実行してもらいました。 次は、Jacob の場合と同じ分析を Kerry の Loop ページに対して実行してください。 これを行うには、Kerry に対して手順 4 から 7 を繰り返します。

11.  次に、"**Contoso マネージャー育成計画**" ワークスペースの下に新しいページを追加します。 このページのタイトルを「**アクション プランの要約**」に変更します。

12.  この新しいページの Copilot ペインを開きます。 次に、このワークスペースの各 Loop ページのアクション プランを確認し、潜在的な課題、必要なサポート、進行状況の追跡方法など、各マネージャーのアクション プランを実際の組織でどのように実装できるかを要約するよう Copilot に依頼します。

13.  結果を確認し、その内容をコピーして「**アクション プランの要約**」ページに貼り付けます。 アイデアと一緒にコピーし、貼り付けた余分なテキストを削除します。

マネージャー育成計画の中心ハブとして機能する共同作業用の Loop ワークスペースを作成しました。 このタスクにより、Loop と Copilot を共同ツールとして使用し、現代の人事環境でマネージャーの育成計画を促進するという実践的な経験を積むことができました。
