# 演習 1、タスク 4: Copilot Studio を使って、規制調査アシスタントを作成する
---
監査ウィンドウが開くと、規制調査が流れ始めます。最初は定期的な明確化、次に文書のサポートの要求、それからデータ処理義務の詳細な説明を求める突然のフォローアップです。 Boulder Innovation は、これらの日常的なチェックがどれほど早くエスカレートしてあふれかえる場合があるかを知っています。

あなたは、法務チームが先手を打てるようにするため、必要に応じて Web で当局の規制ソースを検索できる Copilot Studio エージェントを作成するよう求められています。 正式なガイダンスを探すために何時間も費やす代わりに、チームは、カリフォルニア州消費者プライバシー法 (CCPA) とカリフォルニア州プライバシー権法 (CPRA) に関連する州監査に関連する新しい質問が受信トレイでヒットするたびに、スマートで信頼できる調査アシスタントを利用します。

> [!NOTE] 
> この演習では、Copilot Studio Lite エクスペリエンスを使って、規制調査アシスタント エージェントを作成します。 このシンプル エクスペリエンスは、普通のビジネス ユーザー向けに設計されており、プログラミング スキルは必要ありません。 一方で、より複雑で高度なエージェントを構築するソフトウェア開発者は、通常、Copilot Studio Full エクスペリエンスを使用します。

このタスクを完了するには、次の手順を実行します。

1.  Microsoft Edge ブラウザーで新しいタブを開き、Microsoft 365 を開きます。

2.  Microsoft 365 のナビゲーション ウィンドウで **[新しいエージェント]** を選択します。 これにより、Copilot Studio の**エージェント ビルダー**が開き、**[新しいエージェント]** ページが表示されます。

3.  **[新しいエージェント]** ページで、エージェントを作成するよう Copilot に指示します。 プロンプトでは、エージェントの名前と、エージェントの概要、対象ユーザー、それで実行したいことの一般的な説明を入力する必要があります。  
    <br/>このエージェントの場合は、次のプロンプトを入力し、進む矢印 (送信) アイコンを選択して、プロンプトを送信します。  
    <br/>**規制調査アシスタントというタイトルのエージェントを作成します。このエージェントの目的は、CCPA と CPRA に関する当局のガイダンスを Web で検索し、監査関連の質問に対する明確で十分な根拠のある回答を生成することです。このエージェントは、法務チームが迅速、正確、確実に規制当局に対応するのに役立ちます。エージェントは調査と下書き作成の補助であり、法律顧問ではありません。**

4.  進む矢印を選択すると、新しいエージェント用の**エージェント ビルダー** フォームが表示されます。 このフォームの上部に、**[説明]** タブと **[構成]** タブがあります。

    - **[説明]** タブを使用すると、Copilot との会話を続けられます。 このタブは既定で表示されます。

    - **[構成]** タブでは、エージェントを動作させるための詳細な設定を定義できます。

    Copilot がエージェントを作成するまで 1 から 2 分待つと、エージェントの名前と説明が **[エージェント プレビュー]** ペインに表示されます。

5.  フォームの上部にある **[構成]** タブを選びます。 入力したプロンプトに基づいて Copilot が行ったことを見てみましょう。

6.  **[構成]** タブの **[名前]** と **[説明]** フィールドが、入力したプロンプトに基づいて設定されているはずです。 **[指示]** フィールドまで下スクロールします。 Copilot は、あなたが最初のプロンプトで指定した説明に基づいて、これらの指示を生成しました。 Copilot によって生成された詳細レベルの指示を確認します。

   > [!IMPORTANT]
   > エージェント ビルダーのプロセスが優れている点は、ユーザーによる基本的な自然言語の説明が、Copilot によって自動的に複雑な指示のセットに変換されることです。 このプロセスにより、ユーザーは詳細な指示のセットを自分で作成する必要がなくなります。

7.  指示を変更したい場合は、**[指示]** フィールドで直接手動で編集することも、代わりに指示を更新するよう Copilot に依頼することもできます。  
    <br/>あなたは、**[指示]** を確認した後、Copilot に他のいくつかの項目を指示セットに追加させることにしました。 これを行うには、**[説明]** タブを選んで、次のプロンプトを入力します。

    **指示を更新して、次の項目を含めます。回答は、専門的で正確で中立的な法的トーンで記述する必要があります。エージェントは、不確実なこと、バージョンの変更、または関連性がある保留中の規則制定にフラグを設定する必要があります。**

8.  指示を更新した後、Copilot の応答を確認します。 Copilot が行った変更を確認するには、**[構成]** タブを選び、**[指示]** フィールドまで下にスクロールします。 要求した 2 つの新しい指示が Copilot によって追加されたことを確認します。

9.  この時点の指示は問題なさそうに見えますが、まだ改良できるかもしれないとあなたは考えます。 どのように改良すればよいかよくわからないため、Copilot に意見を聞くことにします。
    <br/><br/>そのためには、**[説明]** タブを選びます。今度は、このエージェントを改良できる他の指示を提案するよう Copilot に依頼するプロンプトを入力します。

10.  Copilot の推奨を確認します。 その提案に満足したので、エージェントの指示にそれらをすべて追加するよう Copilot に依頼します。

11.  指示を更新したことを Copilot が応答したら、**[構成]** タブを選んで **[指示]** までスクロールします。 Copilot が追加した新しい項目に注意します。

12.  これで、指示は満足できるものになり、エージェントのナレッジ ソースとスターター プロンプトを構成する準備ができました。  
    <br/>**[構成]** タブで **[ナレッジ]** セクションまで下にスクロールし、**[すべての Web サイトを検索する]** トグル スイッチが有効になっていることを確認します。 Copilot は、元のプロンプトで指定された説明 (つまり、"**このエージェントの目的は、...を Web で検索し**...") に基づいてエージェントを作成したときに、このトグル スイッチを有効にしているはずです。 トグル スイッチが有効になっていない場合は、ここで有効にします。

13.  **[推奨プロンプト]** では、Copilot にプロンプトを生成させることも、独自のプロンプトを手作りすることもできます。 両方の方法を試してみましょう。  
    <br/>Copilot に推奨プロンプトを生成させるには、**[説明]** タブを選び、エージェント用の推奨プロンプトを 3 つ生成するよう Copilot に依頼します。 各プロンプトにどのようにタイトルとメッセージが作成されるかに注意してください。

14.  次に、独自のプロンプトをいくつか入力します。 **[構成]** タブを選び、**[推奨プロンプト]** セクションまで下にスクロールします。 Copilot がエージェントに追加した 3 つのプロンプトが表示されます。  
    <br/>手作業で追加するプロンプトごとに、プロンプトの下に表示される **[推奨プロンプトを 1 つ追加する]** オプションを選びます。  
    <br/>一般的な規制トピックに関連する 6 つの推奨プロンプトが下に表示されます。 これらのプロンプトを確認し、気に入ったプロンプトを 2 つか 3 つ選んで、エージェントに追加します。

        - **タイトル:** 消費者権利タイムライン
            - **メッセージ:** CCPA/CPRA (アクセス、削除、修正) で検証された消費者要求にはどのような応答タイムラインと検証要件が適用され、どのような例外によって履行が制限される可能性がありますか?  
                
        - **タイトル:** 共有と販売の定義と影響
            - **メッセージ:** CPRA では個人情報の "共有" と "販売" がどのように定義および区別され、オプトアウトの権利とクロスコンテキストの行動ターゲティング広告に対する実際的なコンプライアンスの影響はどのようなものですか?  
                
        - **タイトル:** 侵害通知の基本
            - **メッセージ:** カリフォルニア州の住民に影響を与える侵害通知の主要な要素、タイムライン、受信者の要件と、規制の通信で引用する必要がある権限のあるソースは何ですか?  
                
        - **タイトル:** データ保持と最小化のガイダンス
            - **メッセージ:** データ保持、最小化、目的の制限に関する現在の CPRA の要件を、法的レビューに適した平易な言葉で要約します。 開示の期待に関する簡潔な説明を提供し、保持スケジュールを構築または更新するために企業が実行する必要がある 5 つの運用手順の一覧を示します。 当局の Web ソース (カリフォルニア プライバシー保護機関、法令ページなど) を優先し、最新のガイダンスへの直接リンクを付けた引用を含めます。  
                
        - **タイトル:** サービス プロバイダー/請負業者の義務
            - **メッセージ:** CCPA/CPRA でのサービス プロバイダーと請負業者に対する契約上の義務の概要を示し、必要な条項 (使用制限、消費者要求に対する支援、転送制御、監査/評価) とベンダー管理に対する実用的な影響について説明します。 テンプレートの下書き作成を支援し、関連する法定テキストへのリンクを付けて当局のソース (.gov または公式機関のガイダンス) を引用するため、"条項名 → 目的" の簡単な表を提示します。  
                
        - **タイトル:** 適用と罰則リスクの概要
            - **メッセージ:** カリフォルニア プライバシー保護局と司法長官の役割に関する CCPA/CPRA の下での執行メカニズムと罰則について説明する簡潔なリスク概要を作成します。 3 つの一般的な非コンプライアンス シナリオ (オプトアウトの遵守の失敗、消費者要求の不適切な処理、不十分な開示) に関するリスク マトリックス (低/中/高) を含め、推奨される是正措置を追加します。 当局の Web ソースでの概要を典拠とし、直接リンクを含む引用を提供します。  
            
15.  推奨プロンプトをいくつかテストします。 エージェントが応答ごとに引用とリンクを含めることを確認します。

16.  推奨プロンプトの結果に問題がなければ、**[作成]** ボタンを選んでエージェントを作成します。

17. エージェントが作成されると、エージェントが正常に作成されたことを示すダイアログ ボックスが表示されます。 このダイアログ ボックスでは、エージェントに移動したり、それを共有したりできます。 **[エージェントに移動する]** オプションを選択します。

> [!NOTE]
> この段階では、エージェントは非公開であり、アクセスできるのは自分だけです。 エージェントを複数のチーム メンバーが使う必要がある実際のシナリオでは、それらのユーザーと共有します。 このトレーニング演習では、自分専用のテナント内で作業しているため、共有は必要ありません。
