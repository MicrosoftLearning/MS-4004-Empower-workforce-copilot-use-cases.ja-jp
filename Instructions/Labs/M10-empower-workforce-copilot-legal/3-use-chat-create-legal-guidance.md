# 演習 1、タスク 1: Copilot Chat を使って、規制の法的ガイダンスを作成する
---
Boulder Innovations の共同弁護士であるあなたは、法務チームによる CCPA と CPRA に関する内部ガイダンス概要の下書き作成のサポートを担当しています。 あなたの役割は、CCPA と CPRA のコンプライアンスに関する Boulder Innovations の準備状況がわかる 2 つの重要な成果物を生成して、法務部門をサポートすることです。 調査と下書き作成のプロセスを効率化するため、あなたは、規制の調査、および法務と法務以外の両方の部門の対象ユーザーのための明確で実用的な内容の生成に、Microsoft 365 Copilot Chat を使おうと考えています。

2 つの主な目的は次のとおりです。

- CCPA と CPRA 両方の主要な規定を、Boulder Innovations のビジネス モデルと運用にそれが具体的にどのように適用されるのかに焦点を当てて要約します。 この要約は法務部門を対象としたものであり、関連する規制の定義、基準、義務、適用リスクを含む必要があります。

- IT、マーケティング、製品開発の部門長向けに調整された、法律概要の下書きを作成します。 このドキュメントでは、会社が法令遵守を達成するために各チームが次に実行しなければならない手順について説明されている必要があります。 法律の専門用語を避け、法務部門以外の利害関係者が自分の責任と予定を簡単に理解できるように、平易でわかりやすい言葉で書きます。

このタスクを完了するには、次の手順を実行します。

1.  Microsoft Edge ブラウザーで **Microsoft 365** のホーム ページ **(https://www.microsoft365.com)** にサインインします。 Copilot Chat で CCPA と CPRA の Web コンテンツにアクセスしたいので、**[Web]** 切り替えスイッチを選びます。

2.  CCPA と CPRA の法令の重要な規定を要約するよう Copilot に指示します。

   > [!NOTE]
   > この最初のプロンプトでは、概要ユニットで説明した 4 つの重要な要素が組み込まれている場合に、効果的なプロンプトがどのようになるかわかるよう、こちらでテキストを用意しました。 この演習の残りのプロンプトはすべて学習者が記述する必要がありますが、その際は、このプロンプトを手本として参考にできます。
    
   **私は、米国の玩具製造販売会社である Boulder Innovations の共同弁護士です。カリフォルニア州消費者プライバシー法 (CCPA) とカリフォルニア プライバシー権法 (CPRA) の主要な規定の概説を作成してください。このドキュメントは法務部門を対象としており、関連する規制の定義、適用基準、企業の義務、実施メカニズム、潜在的なリスクを含む必要があります。この要約では、これらの法律が米国居住者の個人情報を収集、保存、処理する Boulder Innovation の能力にどのように影響するかに焦点を当てる必要があります。**

3.  Copilot Chat によって作成された要約をレビューします。 それは良いたたき台であると思われますが、IT と法務部門による保持スケジュール、削除手続き、ユーザー データ アクセス システムの確立をサポートするために、さらに多くのことを行う必要があることが懸念されます。 この欠点に対処するには、CCPA と CPRA が Boulder Innovations におけるデータの保持と削除のポリシーに与える影響をまとめた別のセクションを追加するよう Copilot に指示します。 Boulder がコンプライアンスを確保できる方法に関する推奨事項を提供するよう、Copilot に指示します (特に、データの削除、修正、アクセスなどのユーザー権限に関するもの)。

4.  次に、Copilot Chat を使って、プライバシーを考慮した設計を実践し、異なる部署にも認識を持たせます。 そのためには、新しいデジタル製品を最初から CCPA と CPRA に準拠させるのに役立つ、Boulder Innovations の製品開発チーム向けのアクション プランを提供する別のセクションを追加するよう、Copilot に指示します。

5.  これで、ドキュメントの範囲については満足できるものになりました。 ただし、クロスウォーク テーブルを作れば、法務担当者に役立つ可能性があります。それを使えば、変更点や注意し直す必要がある箇所をすばやく確認できます。 そのためには、CCPA と CPRA の規定を並べて比較し、CPRA での新しい義務または拡張された義務がはっきりわかるテーブルをこのドキュメントに作成するよう、Copilot に指示します。

6.  テストでは、Copilot によってチャット ページに比較テーブルが表示されたり、テーブルを含む別のドキュメント (そのドキュメントへのリンクが提供されます) が作成されたりすることがありました。 ドキュメントへのリンクが提供される場合は、リンクを選んでダウンロードされたドキュメントを開き、正しいことを確認します。

7.  これで、この概要ドキュメントは完全なものになったと思われるので、このイニシアティブに割り当てられている法務部門のプロジェクト チームのメンバーに配布するために保存します。 Copilot Chat の応答はプロンプトごとに個別に作成され、1 つのドキュメントに結合されていなかったので、次のそのようにしようと思います。 これまでの応答で生成したすべての情報を、ダウンロードできる 1 つのドキュメントにまとめるよう、Copilot に指示します。 比較テーブルを含めるようにします。

8.  Copilot Chat は、以前の応答のすべての情報が結合された 1 つのドキュメントを生成するはずです。 これで、このプロジェクトの 2 番目の要件に対応できるようになりました。

9.  次に、Copilot を使って、IT、マーケティング、製品開発の部門長向けに調整された法律概要の下書きを作成します。 このドキュメントでは、会社が法令遵守を達成するために各チームが次に実行しなければならない手順について説明されている必要があります。 この法律概要を作成するには、会社が CCPA と CPRA に準拠するために、Boulder Innovations の IT、マーケティング、製品開発チームが実施する必要のある次の手順が要約された法律概要を作成するよう、Copilot に指示します。 この法律概要は、これら 3 つの各部署の長に合わせて調整する必要があります。 法務以外の関係者が自分の責任を簡単に理解できるように、平易でわかりやすい言葉を使い、法律の専門用語を避けるよう、Copilot に指示します。 実際的なアクション、推奨されるタイムライン、各部署の作業によって全体的な法令遵守がどのようにサポートされるかに焦点を当てるよう、Copilot に指示します。

10.  Copilot で生成された法律概要を確認します。 この要約では次の手順の概要が示されていますが、2 つの点を更新するといっそう効果的になると思われます。 次の要求を含む 1 つのプロンプトを送信します。

        - まず、法律概要を実用的な計画ツールに変換します。 そのためには、既存のドキュメントを更新し、次の 30 日、60 日、90 日間に完了する必要があるアクションを示す簡単な法令遵守タイムラインまたはチェックリストを追加するよう、Copilot に指示します。
    
        - また、会社がこれらの規制に準拠していない場合のリスクを利害関係者に理解させる必要があります。 緊急性を強調し、これらの手順が重要である理由を利害関係者が理解するのを助けたいと思います。 そのためには、平易な言葉を使って、CCPA と CPRA に準拠しない場合の潜在的なリスクについて説明する短いセクションを含めることで、この法律概要をより良くするよう、Copilot に指示します。
    
        - これらの要求の両方を法律概要に適用し、ダウンロード可能なドキュメントにするよう、Copilot に指示します。

11.  更新されたドキュメントへのリンクを選び、Copilot Chat によって 2 つの新しいセクションが追加されたことを確認します。

12. 最初の指示では、IT、マーケティング、製品開発を対象とする法律概要を作成しましたが、部門ごとに部門固有の要件を含む 3 つのドキュメントを個別に作成すれば、いっそう効率的かもしれません。 そのようにすると、すべての部署が 1 つのドキュメントにまとめられている、Copilot Chat が作成した最初の法的準備書面を、上級管理者向けにすることができます。 逆に、部署ごとの個別の法的準備書面では、部門長は、余分なコンテキストなしに、必要な情報だけを入手できます。  
    <br/>Boulder のマーケティング部門だけを対象とするダウンロード可能な法律概要を作成するよう、Copilot に指示します。 この法律概要では、CCPA と CPRA の規制が、デジタル広告、顧客データ収集、ユーザーの同意手続きなどのマーケティング サービスにどのように影響するかを指定する必要があります。

13. マーケティングの法律概要をダウンロードして確認します。 要求したすべての情報がドキュメントに含まれることを確認します。

14. 次に、Boulder の IT 部門だけを対象とする法律概要を作成するよう、Copilot に指示します。 この法律概要では、CCPA と CPRA の規制が、データ セキュリティ、ユーザー アクセス要求、データ削除プロトコル、システム ログなどの IT の領域にどのように影響するかを指定する必要があります。 技術面で規制に確実に準拠するための実用的な推奨事項を提供するよう、Copilot に指示します。

15. IT の法律概要をダウンロードして確認します。 要求したすべての情報がドキュメントに含まれることを確認します。

16. 最後に、Boulder の製品開発だけを対象とする法律概要を作成するよう、Copilot に指示します。 この法律概要では、CCPA と CPRA の規制が、プライバシー対応設計の原則、既定のデータ収集設定、新しいデジタル製品での同意メカニズムなど、製品開発の分野にどのように影響するかを指定する必要があります。 製品が初期設計から発売までプライバシー規制に確実に準拠するための推奨事項を提供するよう、Copilot に指示します。

17. 製品開発の法律概要をダウンロードして確認します。 要求したすべての情報がドキュメントに含まれることを確認します。
